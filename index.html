Shoky LMS

Featured Education · LMS

Shoky LMS

A full-featured Learning Management System designed to make course creation and delivery simple, fast, and scalable — without the enterprise weight that slows everyone down.

Year2026
StatusProduction
Duration4 months
RoleLead Architect
  • Node.js
  • React
  • PostgreSQL
  • Docker
shoky.lms/dashboard

The Problem

Most LMS platforms are either over-engineered for large institutions or too limited for independent educators. They're slow to configure, expensive to maintain, and designed around outdated pedagogical models that assume lecture-first, learner-last. Educators with real expertise needed a tool built for them — not repurposed from an enterprise system.

The Solution

Shoky LMS was built ground-up with the educator as the primary user. Clean course management. Progress tracking that actually helps learners. Assessment tools that respect both educator time and student attention. A content delivery system that stays out of the way. The interface is deliberately minimal — because the content should be the focus, never the platform.

Architecture & Stack

  • Node.js
  • React
  • PostgreSQL
  • REST API
  • Docker
  • Linux
  • Nginx

Backend handles authentication, course data, progress tracking, and content delivery via a clean REST API. Frontend delivers a focused educator and student interface. Containerized with Docker for consistent, environment-agnostic deployment. Runs natively on Linux.

Impact & Lessons Learned

Shoky LMS removes the barrier between expertise and education. An educator with knowledge but no platform can now be fully operational in hours, not months. The core lesson: great educational software accelerates teaching — it never complicates it. Every design decision should ask "does this help the student learn?" before "does this look impressive?"

Automation · WhatsApp · Bots

WhatsApp Automation

An automation engine for WhatsApp that handles repetitive communication tasks — responses, workflows, notifications — without requiring human intervention for every exchange.

Year2025
StatusOpen Source
Duration2 months
RoleDevOps Eng.
  • Python
  • WhatsApp API
  • Webhooks
  • Docker

The Problem

Businesses and individuals managing high-volume WhatsApp communication spend enormous time on repetitive tasks: answering the same questions, sending scheduled messages, routing inquiries. Manual handling doesn't scale — and every repeated message is wasted human attention that could go toward something that actually requires judgment.

The Solution

A programmable automation engine that sits between WhatsApp and the user's workflow. Define rules, triggers, and responses — the system handles the routine. Humans stay in the loop for what actually requires them. Configurable without engineering expertise. Deployable without infrastructure complexity.

Architecture & Stack

  • Python
  • WhatsApp Business API
  • Webhooks
  • Docker
  • Linux

Python handles automation logic and message parsing. Webhooks bridge WhatsApp events to the processing engine in real time. Containerized for reliable, repeatable deployment on any Linux server. No external dependencies beyond the API itself.

Impact & Lessons Learned

The biggest lesson: automation is most valuable when it respects human time. Knowing which interactions to automate and which to preserve for humans is the real engineering challenge — not the code. The hardest part of automation isn't writing the script. It's designing the boundaries.

Chrome Extension

Browser · Extension · Productivity

Chrome Extension

A purpose-built Chrome Extension that enhances browser workflow using Manifest V3 — modern, secure, and focused on doing exactly one thing well.

Year2025
StatusExperimental
Duration1 month
RoleExtension Dev
  • JavaScript
  • Chrome APIs
  • Manifest V3
  • Service Worker
workspace/projects

The Problem

The browser is the most-used interface in modern workflows, yet off-the-shelf extensions almost always do too much or too little. They bloat permissions, request access far beyond their stated purpose, and age poorly as the extension API evolves. The right solution for a specific workflow problem is a purpose-built tool — not a generic one adapted to fit.

The Solution

A focused Chrome Extension built to Manifest V3 standards. One specific use case, solved completely, with minimal permissions. Fast, transparent, and maintainable. Built to the modern extension architecture that treats security as a first-class constraint, not an afterthought.

Architecture & Stack

  • JavaScript
  • Chrome Extension API
  • Manifest V3
  • Content Scripts
  • Service Worker
  • Chrome Storage API

Built on Manifest V3 for security compliance and modern browser compatibility. Content scripts interact with page DOM where needed. Service worker handles background logic without persistent overhead. Zero unnecessary permissions.

Impact & Lessons Learned

Building browser extensions teaches a crucial engineering discipline: the security model is the architecture. Manifest V3 restrictions don't limit the extension — they force better decisions. Understanding why a permission exists before requesting it is the difference between engineering and hacking together something that works. The lesson applies far beyond extensions.
